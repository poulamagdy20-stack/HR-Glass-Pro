# HR Glass Pro - Payroll Payment Review

تمت مراجعة وتعديل نسخة V19 Crystal UI مع إضافة دفع وتسوية مرتبات الموظفين.

## المضاف في قسم المرتبات
- زر Pay / Settle لكل موظف.
- نافذة تسوية احترافية تعرض:
  - صافي الحساب الحالي.
  - المبلغ المدفوع.
  - المتبقي بعد الدفع.
  - طريقة الحساب: Daily / Weekly / Monthly / Hourly.
- الدفع الكامل يجعل صافي حساب الفترة = 0.
- الدفع الجزئي يترك المتبقي ظاهرًا في كارت المرتب.
- كل عملية دفع تحفظ في collection: `transactions` بنوع `salary_payment`.
- الطباعة أصبحت تعرض المدفوعات ضمن كشف الموظف.

## طريقة الحساب
- Daily: يحسب مرتب يوم واحد بناءً على التاريخ المختار.
- Weekly: يحسب أسبوع التاريخ المختار.
- Monthly: يحسب شهر التاريخ المختار.
- Hourly: يحسب حسب ساعات العمل الفعلية المسجلة في الشهر وسعر الساعة.

## المراجعة الفنية
- تم فحص syntax لملفات:
  - js/main.js
  - js/payroll.js
  - js/i18n.js
- لم يتم تغيير إعداد Firebase الخاص بالمشروع.
- Firestore Rules الحالية تسمح للأدمن بإنشاء transactions، وهذا مطلوب لعملية الدفع.

## التشغيل
شغل المشروع من localhost وليس file://:

```powershell
npx serve .
```

ثم افتح الرابط الذي يظهر مثل:

```text
http://localhost:3000
```
